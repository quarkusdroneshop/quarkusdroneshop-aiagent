1. Purpose
-----------------------------------------------------------

本システムはAI Agentを利用して自然言語によるデータ登録を実現する。

AI AgentはOpenMetadataからメタデータを取得し、
対象データのスキーマやビジネスルールを理解する。

不足している項目は利用者へ対話形式で問い合わせる。

必要な情報が揃ったらBusiness APIへデータ登録を依頼する。

Business APIはQuarkusで実装し、
業務ロジック、トランザクション、永続化を担当する。

===========================================================
Responsibilities
===========================================================

AI Agent

Responsibilities

- User interaction
- Natural language understanding
- Schema lookup
- Required field detection
- Data type validation
- Business rule confirmation
- Business API invocation

AI Agent must NOT

- Execute SQL
- Access database directly
- Execute business logic
- Maintain transactions



-----------------------------------------------------------

OpenMetadata

Responsibilities

- Table definitions
- Column definitions
- Business glossary
- Domains
- Owners
- Data quality rules
- Required columns
- Descriptions

OpenMetadata is the single source of truth for metadata.


-----------------------------------------------------------

Business API (Quarkus)

Responsibilities

- REST API
- Bean Validation
- Business validation
- Transaction management
- Persistence
- Audit logging
- Authorization
- Event publishing

Business API must NOT

- Call LLM
- Implement AI logic
- Interpret natural language

===========================================================
Design Principles
===========================================================

1. OpenMetadata manages metadata.

2. AI Agent performs reasoning.

3. Quarkus executes business logic.

4. Database is accessed only through Business API.

5. AI Agent never executes SQL.

6. Metadata is never duplicated.

7. Business rules belong to Business API.

8. AI is stateless.

9. Components communicate through REST APIs.

10. The architecture must remain cloud-native and deployable on OpenShift.
